## 2026-03-09 - Mass Assignment in Create Functions
**Vulnerability:** Mass assignment vulnerabilities found in `createContact`, `createTask`, and `createDeal` where an untrusted data payload was spread at the end of `.insert().values({ ...data })` statements. This allowed an attacker to overwrite the server-generated `id` and `organizationId` with arbitrary values, potentially taking ownership of records or bypassing authorization.
**Learning:** `create` functions that accept `Record<string, unknown>` payloads and spread them using object spread syntax (`...`) are vulnerable to mass assignment if the untrusted payload is spread *after* or alongside sensitive fields without explicit omission.
**Prevention:** To prevent mass assignment vulnerabilities when handling inserts and updates, explicitly destructure and remove sensitive fields (`id`, `organizationId`) from the incoming payload (e.g., `const { id: _id, organizationId: _orgId, ...safeData } = data.data;`). Always spread the untrusted `safeData` first, then explicitly assign trusted, server-generated values (like `id` and `organizationId`) to ensure they take precedence and cannot be overwritten.
